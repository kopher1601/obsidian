에러 메세지 등에 예외 스택을 출력해서는 안된다. 종속성의 버전을 통해 취약점을 알아내 공격당할 수 있기 때문이다. 취약한 종속성은 이용하지 않는 것이 최선이지만, 실수로 취약한 종속성을 이용했더라도 이를 광고할 필요는 없다.

또한, 클라이언트로 반환되는 응답이 특정 입력이 무엇인지 추측하게 도와줘서는 안 된다.
```json
// 응답 A (특정 입력이 무엇인지 추측하게됨)
{
	"status": 401,
	"error": "UnAuthorize",
	"message": "사용자 이름이 올바르지 않음",
	"path": "/login"
},
// 응답 B (특정 입력이 무엇인지 추측하게됨)
{
	"status": 401,
	"error": "UnAuthorize",
	"message": "암호가 올바르지 않음",
	"path": "/login"
}

// 올바른 반환
{
	"status": 401,
	"error": "UnAuthorize",
	"message": "사용자 이름 또는 암호가 올바르지 않음",
	"path": "/login"
}
```
