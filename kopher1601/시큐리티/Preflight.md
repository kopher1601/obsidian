Preflight는 CORS 메커니즘의 중요한 부분으로, 브라우저가 실제 요청을 보내기 전에 서버가 해당 요청을 허용하는지 확인하는 예비 요청입니다. 이 요청은 HTTP `OPTIONS` 메서드를 사용하여 전송됩니다.

**프리플라이트 요청은 다음과 같은 경우에 발생**합니다:
- 단순하지 않은 요청 메서드(`PUT`, `DELETE` 등) 사용 시
- 특정 헤더(`Content-Type: application/json` 등)가 포함된 경우
- 사용자 정의(커스텀) 헤더가 포함된 경우
**서버는** 이 Preflight 요청에 응답하여 허용되는 메서드, 헤더, 도메인 등의 정보를 제공해야한다. 브라우저는 이 응답을 확인한 후 실제 요청을 보낼지 결정합니다.
