인증은 애플리케이션이 사용자(사람 또는 다른 애플리케이션)을 식별하는 방법이다. 

인증 취약성이 있다는 것은 사용자가 악의를 가지고 다른 사람의 기능이나ㅏ 데이터에 접근할 수 있다는 의미다.

## 세션 방식
세션 객체를 통해 인증을 처리하는 방식
![[Pasted image 20251107144108.png]]
- **장점** : 세션 객체를 이용해, 사용자들의 상태를 관리할 수 있다
- **단점** : 메모리를 과하게 사용, 데이터베이스를 자주 조회해야 함

## 토큰 방식
[[JWT]] 토큰을 이용해 인증을 처리하는 방식
![[Pasted image 20251107151403.png]]
- **장점** : 데이터베이스 조회 필요가 없다
- **단점** : 사용자들의 상태를 관리할 수 없다. 또한 토큰 탈취시 어떤 조취도 취할수가 없다
